# 잘했다고 생각하는 부분

위스키 관리 프로그램에서 Python 패키지를 직접 구성하고, `Alcohol`, `Whisky`, `WhiskyLibrary` 클래스를 나누어 역할을 분리한 부분이 잘했다고 생각한다. `Whisky` 클래스는 `Alcohol` 클래스를 상속하여 공통 속성인 이름, 도수, 용량, 가격을 재사용하고, 위스키만의 속성인 국가와 종류를 추가하였다.
이를 통해서 위스키를 국가별로 검색 가능하고 위스키의 종류가 무엇인지 확인도 가능하다.

또한 pytest 테스트를 작성하여 위스키 추가, 삭제, 검색, 평균 도수 계산, 총 가격 계산, 잘못된 도수 입력 같은 상황을 검증하였다. 함수를 정의하고 실제로 사용하였으나 너무 정형화된 방식이었기에 새로 만든 부분이 마음에 들었다.
# 아쉬운 부분

처음에는 위스키 종류를 분류하는 방식이 단순 함수로 정의하여 코드가 반복되고 종류 구분도 정확하지 않았다. 
따라서 코드와 테스트 방식과 검증 범위를 새로 만드느라 시간을 낭비했다.
또한 README의 설치 방법과 테스트 결과가 실제 프로젝트 상태와 완전히 일치하지 않았다. 또한 pycodestyle 경고가 남아 있어서 제출 기준을 다시 확인하고 수정할 필요가 있었다. 독스트링 작성도 ai에게 기본 틀을 받아 만들어 보았으나 너무 조악한 수준이어서 전부 바꾸느라 시간과 노력이 불필요하게 들어갔다.

기능 면에서는 위스키 정보를 메모리의 리스트로만 관리하기 때문에 프로그램을 종료하면 데이터가 유지되지 않는 점이 아쉬웠다. JSON이나 CSV 저장 기능이 있었다면 더 실용적인 패키지가 되었을 것 같다.

# 다음에 개선하고 싶은 점

다음에는 위스키 컬렉션을 다시 불러오는 기능을 추가하고 싶다. 또한 이름 검색, 종류별 검색, 가격순 정렬, 도수 범위 필터링 같은 기능을 추가하면 실제 컬렉션 관리 도구로 더 쓸모가 있을 것 같다. 마지막으로는 이런 기능들을 전부 합치고 UI를 만들어 앱으로 출시하면 재밌을것 같다.

결국 불필요한 곳에서 시간과 노력을 낭비한것이 아쉬웠기에 이런 프로젝트를 한다면 기본적인 틀을 더 제대로 잡고 시작하고 싶다.

# AI 도구 사용 내역

ChatGPT의 codex와 claude를 사용하여 독스트링을 작성하게 해보았으나 영어로 조악하게 만들어졌기에 독스트링 작성 방법을 참고하여 새롭게 작성하였다. 과제 PDF의 요구사항과 GitHub 저장소 상태를 비교하고, README, 테스트 결과, pycodestyle 경고, 필수 제출물 누락 여부를 점검하였다. AI가 제안한 내용을 검토한 뒤, 이를 적용해보고 불필요한 오류, 개선이 필요한 read.me 설명 부분 등을 직접 개선하고 테스트나 실제 코드를 필요할시 새로 만들었다.

# 프로젝트 구조 설명

프로젝트의 핵심 구조를 설명하자면 `Alcohol`은 공통 주류 정보를 관리하는 부모 클래스이고, `Whisky`는 이를 상속하여 위스키의 국가와 종류를 추가한다. `WhiskyLibrary`는 여러 위스키 객체를 컬렉션으로 관리하며 추가, 삭제, 검색, 평균 ABV 계산, 총 가격 계산, 최고가 위스키 찾기 기능을 제공한다.
실제로 whisky.py의 위스키 상세 정보 부분을 보면 함수를 정의하고 부모 클래스 상속, 반환 값을 생성한다.
결국 전체적으로 보면 정보를 입력 후, 이를 상속하고 값을 반환 후 저장된 값을 토대로 기능을 구현시키는 것이다.
중간에 테스트 코드를 새로 만든 것도 테스트가 실제 코드에 맞게 수정되고 엣지 케이스 검증이 강화된 것이다.
처음 틀은 인공지능으로 돌려보았으나 오류가 겉잡을 수 없이 발생하여 결국은 직접 만들게 되었다.
